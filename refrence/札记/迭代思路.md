# 流萤阅读器后续迭代思路

基于流萤阅读器"**离线优先、私密沉浸**"的核心哲学，后续四个维度的演进路线图：

---

## 一、体验层（即刻可落地）

### 1. 排版引擎精细化
- **竖排模式**：实现 CSS `writing-mode: vertical-rl` 切换，适配古籍/轻小说阅读（日式轻小说常有竖排需求）
- **字体本地化**：提供"下载字体包到本地"选项（如方正屏显雅宋、思源宋体），而非仅依赖 Google Fonts（隐私+离线可用）
- **两端对齐优化**：对中文实现标点挤压（punctuation kerning），避免行尾 `。` `，` 单独成行
- **阅读统计**：本地记录每日阅读时长/字数，生成周视图（纯本地 JSON，不上传）

### 2. 交互微创新
- **物理翻页动画**：CSS 3D 变换模拟真实书页卷曲（移动端滑动手势的延续）
- **专注力模式**：点击"沉浸"按钮后，隐藏所有 UI（包括滚动条），仅保留文字，按边缘唤出控件
- **双栏对照**：针对学术文献，实现左右分栏对照阅读（适合论文/双语书籍）

---

## 二、功能层（3-6 个月）

### 3. 知识管理系统（保持本地）
```javascript
// 潜在数据结构
annotation: {
  type: 'highlight' | 'note' | 'bookmark',
  text: '选中的文本',
  note: '用户批注',
  tags: ['哲学', '待整理'],  // 本地标签系统
  created: 'timestamp',
  chapter: '章节名'
}
```
- **划线批注**：支持荧光笔高亮（四种颜色），双击添加旁注（侧边栏滑出）
- **标签系统**：为书籍打本地标签（#哲学 #科幻 #待读），书库支持标签筛选
- **笔记导出**：导出为 Markdown/Notion 格式（保留双向链接 `[[章节名]]`）

### 4. 智能化（本地 AI / 可选云端）
考虑到隐私原则，建议**本地模型优先**：
- **端侧摘要**：集成 `transformers.js`（Hugging Face 的 WebGL 加速模型），对选中段落生成离线摘要（模型约 50MB，IndexedDB 缓存）
- **语义搜索**：使用 `MiniLM` 嵌入模型，实现"找相似段落"（基于向量相似度，非关键词匹配）
- **生词本**：自动识别英文/日文生词，点击显示本地词典（可接入 MDict 格式词库）

### 5. 书源扩展（保持去中心化）
- **RSS 转书籍**：订阅 Newsletter/RSS，自动抓取正文转为 EPUB/MD 存入书库（可配合本地 RSS 服务）
- **剪藏插件**：浏览器扩展"发送到流萤"，清洗网页正文后入库（类似 Instapaper 但本地存储）

---

## 三、架构层（长期）

### 6. 跨平台同步（可选 P2P 方案）
既然拒绝中心化云，可考虑：
- **WebDAV 同步**：支持坚果云/OwnCloud 等，自动同步 `library.json` 和阅读进度（端到端加密可选）
- **局域网同步**：在同一 WiFi 下，手机扫码与电脑配对，点对点传输书籍（WebRTC DataChannel）
- **PWA 离线化**：完善 Service Worker，实现"添加到主屏幕"后完全离线可用（无网络也能打开欢迎页）

### 7. 大文件性能优化
- **虚拟滚动 2.0**：当前按章节渲染，可改为**基于字符的虚拟滚动**（仅渲染视口内 ±3 屏内容），支持百万字小说秒开
- **Web Worker 解析**：DOCX/HTML 解析移至后台线程，避免打开大文件时 UI 卡顿
- **增量更新**：修改章节设置时，仅重新渲染变更部分（Virtual DOM diff）

### 8. 无障碍（Accessibility）
- **屏幕阅读器优化**：完善 `aria-label`，让视障用户通过 TTS 功能真正使用
- **高对比度模式**：第五主题，针对视力障碍者（黄底黑字/黑底黄字）
- **键盘导航**：支持完全无鼠标操作（Vim 式快捷键 `j/k` 翻行，`gg/G` 跳转）

---

## 四、生态层（愿景）

### 9. 开源与插件化
- **插件系统**：开放 API 允许第三方开发：
  - 格式插件（支持 EPUB/PDF 解析）
  - 主题插件（CSS 变量注入）
  - AI 插件（本地模型接入规范）
- **书籍推荐算法（本地）**：基于你的书库内容，本地分析后推荐**本地未读的同类书**（基于标题/章节结构的 TF-IDF，不上传数据）

### 10. 实体联动（复古科幻感）
- **E-ink 设备适配**：针对文石/Kindle 浏览器的优化（减少动画，适配 16 级灰度）
- **阅读位置二维码**：生成当前位置的二维码，手机扫码后在手机端继续阅读（通过局域网同步跳转）

---

## 实施建议（分阶段）

**Phase 1（本月）**：划线批注 + 本地标签系统（提升留存）
**Phase 2（季度）**：WebDAV 同步 + 竖排模式（跨设备体验）
**Phase 3（半年）**：本地 AI 摘要 + 虚拟滚动 2.0（技术壁垒）
